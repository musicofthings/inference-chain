Inference Chain

A forward-only n+1 inference ledger for Claude Code. Every session continues sharper than the last.

View on GitHub → Read the PRD/TRD

What it is

A local-first agentic context engineering layer for Claude Code (and, via MCP, Claude Desktop). After each meaningful interaction or session close, the ledger evolves: confirmed beliefs accumulate, rejected ones stay rejected, the frontier sharpens, and the next session starts with a refined operating context — not a transcript.

Session n
→ Session Brief n
→ Memory Evolution Record n
→ Chain Ledger n+1
→ Resume Brief n+1
→ Session n+1 starts sharper

What it is not

not RAG

No embeddings, no vector DB, no retrieval layer.

not a transcript

No chain-of-thought, no tool logs, no terminal capture.

not blockchain

Local SHA-256 hash chain for integrity. That's it.

not SaaS

100% local. No telemetry. No API calls.

Quickstart

pnpm install
pnpm build
pnpm link --global

cd /path/to/your/project
ic init --project-name "My Project"
ic install-claude

In Claude Code

/ic-checkpoint   # mid-session
/ic-stop         # end of session

Back in your terminal

ic ingest .inference-chain/inbox/latest-brief.yml
ic evolve
ic resume

Validate the n+1 hypothesis

Bundled simulation harness replays a scripted multi-session narrative and reports whether the ledger actually carried useful signal forward. 

ic simulate examples/demo-project/build-task-api/sessions \
  --reset --project-name "task-api"

Reports six metrics — anti_repeat_coverage, hypothesis_promotion_rate, frontier_convergence, rejected_persistence, score_progression, final_brief_size_kb — and a binary verdict. On the demo: n+1 POSITIVE (score 0 → 17 over 3 iterations, 0 rejected beliefs reappeared, 75% of do-not-repeat items captured early).

Integrity guarantees

hash chain

Every event carries parentHash + sha256(canonicalJson(event)). ic verify exits non-zero on any tamper.

atomic evolve

SQLite transaction wraps the evolution record, chain state, and events; jsonl is batch-appended only after commit.

canonical JSON

Sorted keys, deterministic bytes. Two processes hashing the same event produce identical hashes.

append-only

No rewrite path. Only way to drop history is to delete .inference-chain/ outright.

Claude Desktop (MCP)

Add to claude_desktop_config.json:

{
  "mcpServers": {
    "inference-chain": {
      "command": "ic",
      "args": ["mcp", "--cwd", "/abs/path/to/your/project"]
    }
  }
}

Exposes six MCP tools: chain_status, chain_resume_brief, chain_ingest_update, chain_ingest_brief, chain_evolve, chain_verify. As long as each session starts in the same folder, the ledger is the key.

Commands

commandwhat it does
ic initInitialize .inference-chain/, SQLite, JSONL, templates
ic install-claudeInstall slash commands + plugin scaffold; merge hooks
ic ingest <file>Validate + store an artifact (routes by kind)
ic evolveApply latest brief/update; emit MemoryEvolutionRecord
ic resumeGenerate resumes/resume_latest.md
ic statusShow iteration, event count, ledger sizes, score
ic verifyReplay hash chain; cross-check SQLite
ic mcpStart MCP stdio server for Claude Desktop
ic simulate <dir>Replay session artifacts + n+1 metrics report
